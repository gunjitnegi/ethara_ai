🗄️ 2. DATABASE DESIGN (MongoDB)
You said: same cluster, two DBs
That’s fine—but honestly unnecessary. Still, here’s how:
DB 1: authDB
users collection
{  _id,  name,  email,  password,  role: "admin" | "member"}

DB 2: appDB
projects
{  _id,  name,  description,  createdBy (adminId),  members: [userId]}
tasks
{  _id,  title,  description,  projectId,  assignedTo,  status: "todo" | "in-progress" | "done",  deadline,  createdBy}

⚠️ Reality Check
Two DBs = more complexity (connections, sync issues).
You gain nothing. One DB is cleaner.
But if they didn’t forbid it, you can still do it.

🔁 4. DATA FLOW (Understand this properly)
Example: Create Task


Frontend → POST /tasks


Backend:


Verify token


Check role = admin


Validate data


Save to DB
Response → success
Example: Member updates task
Frontend → PATCH /tasks/:id
Backend:
Verify token
Check assignedTo === user.id
Update status only
Response
