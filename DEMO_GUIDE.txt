PART 3: HOW RBAC WAS ACHIEVED (The "Technical" part)
------------------------------------------------------------------------
ACTION: Briefly show the code for `auth.middleware.js` or the User model.
SCRIPT: "To achieve this Role-Based Access Control, I implemented a 
three-layer security model:
1. DATABASE: The User schema includes a 'role' field.
2. MIDDLEWARE: I wrote custom Express middlewares like 'verifyToken' 
   and 'adminOnly' that check the JWT before allowing access.
3. FRONTEND: React conditionally renders components based on the 
   role decoded from the token.
I also added IDOR protection so members can't guess IDs to access 
unauthorized projects."

TOP 3 QUESTIONS TO PREPARE FOR:
------------------------------------------------------------------------
1. "How do you handle password security?"
   ANSWER: "We use Bcrypt for one-way salting and hashing. Passwords 
   are never stored in plain text."

2. "How does the server handle both Frontend and Backend?"
   ANSWER: "In production, the Node server serves the 'dist' folder

3. "What happens if a user tries to access an Admin route?"
   ANSWER: "The 'adminOnly' middleware will catch it, return a 403 
   Forbidden status, and log the attempt for security monitoring."
========================================================================
I implemented Role-Based Access Control using custom Express middleware.
The system uses JWT tokens to verify identity. 
Once the identity is verified, my 'adminOnly' middleware checks 
the user's role in the database. 
If a non-admin attempts to access a protected route—like deleting 
a project—the middleware intercepts the request and returns 
a 403 Forbidden status, ensuring that sensitive actions are
only performed by authorized personnel."
